# CORA Security Policy

## Reporting a vulnerability

Open a private security advisory via GitHub: [github.com/coradata/cora/security/advisories/new](https://github.com/coradata/cora/security/advisories/new). This is confidential and visible only to maintainers. Do not file public issues for security disclosures.

A dedicated `security@coradata.org` alias will be published here once provisioned.

## Scope

This policy covers:

- Vulnerabilities in CORA tooling (`/tools/`)
- Vulnerabilities in derivation pipelines that could compromise the integrity of derived artifacts
- Supply chain risks in CORA's CI/CD

Vulnerabilities in hosted upstream standards should be reported to the upstream standards body, not CORA.

## Response

CORA is in bootstrap. Triage happens within 5 business days; resolution timing depends on severity and scope.
